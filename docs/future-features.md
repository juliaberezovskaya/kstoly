# Описание будущих идей, чтобы не забыть ничего

## Черновики рецептов (идея для версии 1.1)

- Система автоматически сохраняет введённые данные рецепта (автосохранение), пока пользователь заполняет форму.
- Автосохранённые данные формируют черновик рецепта, который хранится отдельно от опубликованных рецептов.
- Пользователь может закрыть приложение и продолжить заполнение рецепта позже — черновик восстанавливается при открытии формы.
- Черновик превращается в полноценный рецепт только после явного действия «Сохранить рецепт».
- Если пользователь выбирает «Отмена», черновик удаляется и не появляется в библиотеке.


## Локальное редактирование рецепта в рамках мероприятия (идея для версии 1.1)

**Идея:**  
Организатор может адаптировать рецепт под конкретное мероприятие (убрать ингредиенты, изменить шаги, упростить процесс), не влияя на исходный рецепт в библиотеке. Это позволяет делать “вариации рецептов” для разных событий.

### Механизм работы
- При добавлении рецепта в мероприятие система создаёт **локальную версию (снимок)** рецепта.  
- Организатор может редактировать локальную версию:
  - менять ингредиенты,
  - корректировать количество,
  - менять шаги приготовления,
  - изменять время приготовления.
- Локальные изменения применяются **только к этому мероприятию**.
- Исходный рецепт остаётся неизменным.
- При повторе мероприятия копируется именно локальная версия, включая её изменения.

### Возможность просмотра вариаций
- Для рецептов, использованных в разных мероприятиях, система может показывать список вариаций:
  - название мероприятия,
  - дата,
  - краткое описание отличий от исходного рецепта (например: «− грибы», «+ чеснок», «шаг 3 упрощён»).
- Это позволяет Организатору видеть, как рецепт эволюционировал в разных событиях.

### Дополнительные возможности (опционально)
- Возможность преобразовать локальную версию рецепта в новый рецепт библиотеки ("Сохранить как новый рецепт").
- Возможность сравнить локальную версию с исходной:
  - выделение изменений в ингредиентах,
  - выделение изменений в шагах.
- Возможность сбросить локальную версию и вернуть исходный рецепт.

### Открытые вопросы
- Должна ли локальная версия иметь визуальную пометку (например, «Изменён для мероприятия»)?
- Нужна ли история всех изменений или достаточно финального состояния локальной версии?


## Авторские рецепты и личные копии
- В будущем рецепты могут иметь авторов.
- Организатор не сможет редактировать рецепт, созданный другим пользователем.
- При необходимости он сможет создать личную копию рецепта и изменять её независимо от исходного.

## Корзина для рецептов
- При удалении рецепта он попадает в корзину, а не удаляется физически.
- В списке рецептов пользователь не видит рецепты из корзины.
- Пользователь может просмотреть корзину, восстановить или удалить рецепт окончательно.
- Это позволит безопаснее работать с редкими и важными рецептами.
