# Описание будущих идей, чтобы не забыть ничего

## Черновики рецептов (идея для версии 1.1)
- Автосохранение при вводе данных.
- Возможность продолжить заполнение позже.
- Черновики не появляются в библиотеке до сохранения.
- Пометка “незавершённый”.
- Возможность удалить черновик.
- Ценность: можно временно сохранить недописанный рецепт, но ему придётся проходить валидацию полей. А если мы не готовы к ней, сохранить не даст.

## Локальное редактирование рецепта в рамках мероприятия
**Идея:**  
Дать организатору возможность изменять рецепт только для конкретного мероприятия (например, убрать грибы, сократить шаги, изменить маринад), не создавая новую копию рецепта в библиотеке и не влияя на исходный рецепт.

**Описание:**
- При добавлении рецепта в мероприятие система создаёт локальную копию рецепта (снимок) для этого мероприятия.
- Организатор может редактировать локальную версию:
  - менять ингредиенты (убрать/добавить/заменить),
  - изменять шаги приготовления,
  - корректировать время готовки.
- Изменения применяются только в рамках данного мероприятия.
- Исходный рецепт в библиотеке остаётся без изменений.
- При повторе мероприятия копируется именно локальная версия рецепта (со всеми изменениями).

**Открытые вопросы:**
- Нужно ли явно показывать пользователю, что рецепт “локальный” и основан на исходном?
- Нужна ли возможность «сохранить локальную версию в библиотеку как новый рецепт» (превратить её в постоянный вариант)?

## Авторские рецепты и личные копии
- В будущем рецепты могут иметь авторов.
- Организатор не сможет редактировать рецепт, созданный другим пользователем.
- При необходимости он сможет создать личную копию рецепта и изменять её независимо от исходного.

## Корзина для рецептов
- При удалении рецепта он попадает в корзину, а не удаляется физически.
- В списке рецептов пользователь не видит рецепты из корзины.
- Пользователь может просмотреть корзину, восстановить или удалить рецепт окончательно.
- Это позволит безопаснее работать с редкими и важными рецептами.
