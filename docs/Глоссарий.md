# Глоссарий проекта «К столу!»

Этот глоссарий определяет основные термины, используемые в документации проекта и системе. Он служит единым источником истины (Single Source of Truth) для всех материалов: use cases, требований, UX-описаний и технической документации.

---

## 1. Общие сущности

### **Мероприятие**
Структурированная сущность, объединяющая рецепты, меню, список продуктов и участников. Используется для подготовки к конкретному событию (ужин, праздник, встреча). Содержит информацию о меню, списке покупок, и участниках мероприятия.

### **Меню мероприятия**
Набор рецептов, добавленных пользователем в рамки конкретного мероприятия. Меню определяет, какие блюда будут готовиться в рамках мероприятия.

### **Рецепт**
Структурированная карточка блюда, включающая название, список ингредиентов, шаги приготовления, общее время приготовления, активное время и обложку (список может расширятся в процессе разработки). Может быть добавлен вручную или импортирован из внешних источников.

### **Шаг (процесса приготовления)**
Элемент рецепта, описывающий одну операцию в процессе готовки. Может иметь указания по времени, детализацию действий, используемые ингредиенты.

### **Ингредиент**
Единица списка продуктов. Содержит название, единицу измерения, количество и при необходимости уточнения (тип, бренд, форма упаковки). Может быть объединён автоматически с аналогичными ингредиентами из других рецептов при формировании списка покупок.

### **Список продуктов / список покупок**
Автоматически сформированный перечень ингредиентов, необходимых для рецептов мероприятия. Может редактироваться вручную: добавление, удаление, изменение количества или замена на аналог.

### **Участник мероприятия**
Пользователь, получивший доступ к мероприятию. Может просматривать рецепты, список покупок и помечать купленные продукты. Один участник может иметь доступ к мероприятию с нескольких платформ.

### **Участник по ссылке**
Пользователь, получивший доступ к мероприятию в рамках MVP. Может просматривать рецепты, список покупок и помечать купленные продукты. Участник не имеет аккаунт в рамках MVP. Участники получают доступ к мероприятию по ссылке-приглашению и являются анонимными.

## 2. Функции и процессы

### **Автоматическое объединение ингредиентов**
Процесс, в котором система объединяет одинаковые продукты из разных рецептов, суммируя количество. Пользователь должен понимать, как произошло объединение и иметь возможность посмотреть что конкретно объединилось.

### **Корректировка списка покупок**
Внесение изменений в список продуктов: удаление, добавление, замена ингредиента на аналог или уточнение характеристик. Корректировки отображаются всем участникам.

### **Пошаговый просмотр рецепта**
Режим отображения, в котором рецепт показывается в виде последовательных шагов с указанием времени и подробностей приготовления.

### **Повтор мероприятия**
Процесс создания нового мероприятия на основе ранее проведённого, включая меню, рецепты и список продуктов.

### **Обложка рецепта**
Фотография или миниатюра, прикреплённая к рецепту для визуальной навигации.

### **Импорт рецепта**
Добавление рецепта в систему путём вставки текста, парсинга структурированного контента или ручного ввода.

## 3. Механики взаимодействия

### **Совместная работа**
Функция, позволяющая нескольким пользователям одновременно взаимодействовать с одним мероприятием: отмечать купленное, редактировать список продуктов, просматривать рецепты. Обновления отображаются всем участникам.

### **Доступ по ссылке**
Механизм, позволяющий пользователю присоединиться к мероприятию без регистрации. Ссылка предоставляет ограниченный доступ к контенту мероприятия.

### **Синхронизация изменений**
Обновление данных мероприятия в реальном времени при действиях любого участника (например, отметка «куплено» или корректировка ингредиента).

## 4. Уточняющие термины

### **Общее время приготовления**
Суммарное время, необходимое для выполнения рецепта (включает активное и пассивное время).

### **Активное время приготовления**
Время, которое пользователь непосредственно тратит на выполнение действий (например, нарезка, жарка).

### **Пассивное время приготовления**
Время, которое пользователь не тратит на приготовление (брожение теста, маринование мяса), важное тем, что повар понимает когда начинать приготовление.

### **Аналог продукта**
Продукт, который может заменить основной ингредиент (например, «сыр Пармезан → сыр Грана Падано»).

### **Уточнение к ингредиенту**
Дополнительная информация к продукту: тип салата, сорт мяса, вид пасты, предпочтительный бренд.

### **Скалирование рецепта**
Увеличение или уменьшение количества ингредиентов в соответствии с выбранным количеством порций.

### **Источник рецепта**
Источник создания рецепта: пользователь, ИИ, сайт, заметка. (В MVP не является отдельной сущностью, но важен как термин.)

### **Архивное мероприятие**
Мероприятие, которое находится в архиве. Находится в отдельном месте от основного списка мероприятий.

## 5. Системные термины (для требований и аналитики)

### **БП (Бизнес-проблема)**
Ситуация, возникающая в рамках деятельности организации, которая мешает достижению поставленных целей или приводит к нежелательным последствиям.

### **БЦ (Бизнес-цель)**
Конкретное и измеримое достижение, к которому стремится организация для улучшения своей деятельности, роста и развития

### **БТ (Бизнес-требование)**
Бизнес-требование — это высокоуровневое описание целей и задач проекта, которые необходимо достичь для удовлетворения потребностей бизнеса.

### **UP (User Problem)**
Формализованная пользовательская проблема, являющаяся причиной появления требований.

### **FR (Functional Requirement)**
Функциональное требование, описывающее, что система должна делать.

### **NFR (Non-Functional Requirement)**
Нефункциональное требование: скорость, надёжность, удобство, совместимость.

### **Use Case**
Описание того, как пользователь взаимодействует с системой для достижения цели.

### **Scope MVP**
Минимальный набор функций, необходимых для жизнеспособности продукта.

### **User Story**
Краткое описание функции или требования с точки зрения конечного пользователя.
