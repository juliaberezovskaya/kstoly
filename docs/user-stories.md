# User Stories

## US-R. User Stories для работы с рецептами

### US-R1. Как организатор, я хочу создавать свои рецепты в одном понятном и структурированном месте, чтобы быстро находить их и повторять нужные блюда.
User Problems: UP-1.1, UP-1.2, UP-1.3, UP-1.4, UP-4.1
Use Cases: UC-R1

### US-R2. Как организатор, я хочу иметь возможность отредактировать существующий рецепт, чтобы внести улучшения, которые выявились позднее создания рецепта.
User Problems: UP-1.2, UP-1.4, UP-1.6
Use Cases: UC-R2

### US-R3. Как организатор, я хочу иметь возможность сохранить отредактированный рецепт как новый, чтобы иметь похожий рецепт, но с другим составом.
User Problems: UP-1.6
Use Cases: UC-R2

### US-R4. Как организатор, я хочу иметь возможность посмотреть рецепты в моей библиотеке, чтобы выбрать рецепт для готовки.
User Problems: UP-1.1, UP-1.2, UP-1.3
Use Cases: UC-R3

### US-R5. Как организатор, я хочу посмотреть рецепт из моей библиотеки рецептов, чтобы ознакомиться с его содержанием.
User Problems: UP-1.1, UP-1.2, UP-1.3
Use Cases: UC-R3

### US-R6. Как организатор, я хочу удалить неактуальный рецепт из библиотеки, чтобы он не мешал просмотру актуальных рецептов.
User Problems: UP-1.2, UP-1.3
Use Cases: UC-R4

## US-E. User Stories для мероприятий

### US-E1. Как организатор, я хочу создать мероприятие, чтобы в нём хранилась вся нужная информация в удобной форме с автоматизированными функциями.
User Problems: UP-1.1, UP-1.2, UP-2.1, UP-2.2, UP-5.1, UP-5.2, UP-5.5
Use Cases: UC-E1

### US-E2. Как организатор, я хочу после создания мероприятия сразу перейти к его редактированию, чтобы немедленно продолжить планирование.
User Problems: UP-1.1, UP-1.2, UP-2.1, UP-2.2, UP-5.1, UP-5.2, UP-5.5
Use Cases: UC-E1

### US-E3. Как организатор, я хочу иметь возможность редактировать мероприятие, чтобы изменить информацию о нём и/или управлять рецептами. 
User Problems: UP-1.1, UP-2.1, UP-2.2, UP-5.1, UP-5.2, UP-5.5
Use Cases: UC-E2, UC-E3

### US-E4. Как организатор, я хочу, чтобы при добавлении рецепта в меню ингредиенты автоматически добавлялись в список покупок (и удалялись соответственно), чтобы не пришлось тратить время на ручное оформление списка и уменьшилось количество потенциальных ошибок.
User Problems: UP-2.1, UP-2.2, UP-3.3, UP-3.4
Use Cases: UC-E2

### US-E5. Как организатор, я хочу иметь возможность пересоздать список покупок с нуля, чтобы полностью удалить внесённые вручную правки (для убирания путанницы и возможных ошибок).
User Problems: UP-2.1, UP-2.2
Use Cases: UC-E2

### US-E6. Как организатор, я хочу указать коэффициент порций, чтобы использовать рецепт для большего или меньшего количества гостей.
User Problems: UP-5.4
Use Cases: UC-E2

### US-E7. Как организатор я хочу иметь возможность просмотреть список мероприятий, чтобы получить доступ к информации по моим мероприятиям.
User Problems: UP-5.2
Use Cases: UC-E3

### US-E8. Как организатор я хочу иметь возможность удалить мероприятие, чтобы оно не мешало просмотру остальных.
User Problems: UP-5.2 
Use Cases: UC-E4

### US-E9. Как организатор я хочу переместить мероприятие в архив, чтобы оно не мешало просмотру остальных, но оно возможно понадобилось бы мне позже.
User Problems: UP-5.2
Use Cases: UC-E5

### US-E10. Как организатор я хочу иметь возможность просмотреть список архивных мероприятий, чтобы получить доступ к информации по архивным мероприятиям.
User Problems: UP-5.2
Use Cases: UC-E6

### US-E11. Как организатор я хочу повторить мероприятие, чтобы не тратить время на повторный ввод данных, если мероприятия похожи.
User Problems: UP-5.2
Use Cases: UC-E7


## US-L. User Stories для списка покупок

### US-L1. Как участник приготовления, я хочу просматривать список покупок, чтобы ознакомиться с тем, что необходимо купить
User Problems: UP-3.1, UP-3.4, UP-3.5
Use Cases: UC-L1

### US-L2. Как участник приготовления, я хочу отмечать купленные продукты в списке покупок, чтобы отделять купленное от некупленного.
User Problems: UP-3.1, UP-3.4, UP-3.5
Use Cases: UC-L1

### US-L3. Как участник приготовления, я хочу видеть, из каких рецептов и ингредиентов сформирован каждый пункт в списке покупок, чтобы понимать, откуда он взялся.
User Problems: UP-3.1, UP-3.2, UP-3.3.
Use Cases: UC-L1

### US-L4. Как организатор, я хочу добавлять и удалять пункты в списке покупок, чтобы учитывать дополнительные мелочи, которые не входят ни в один рецепт.
User Problems: UP-2.5, UP-3.5
Use Cases: UC-L1

### US-L5. Как организатор, я хочу иметь возможность пересоздать список покупок на основе текущих рецептов, чтобы сбросить накопившиеся ручные правки и вернуться к базовой версии списка.
User Problems: UP-2.5
Use Cases: UC-L1

## US-C. User Stories для совместной работы

### US-C1. Как организатор, я хочу поделиться доступом к мероприятию, чтобы помощники знали что покупать и что готовить.
User Problems: UP-3.1, UP-3.2, UP-3.3, UP-3.4, UP-4.1.
Use Cases: UC-C1

### US-C2. Как участник мероприятия, я хочу получить доступ к мероприятию по ссылке, чтобы ознакоимиться с рецептом и работать со списком покупок
User Problems: UP-3.1, UP-3.2, UP-3.3, UP-3.4, UP-4.1.
Use Cases: UC-C2
